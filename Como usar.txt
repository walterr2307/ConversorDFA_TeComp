Explicação do Projeto de Manipulação de Autômatos Finitos Determinísticos (AFD)
Visão Geral
Este projeto possui várias classes para gerar, manipular e transformar Autômatos Finitos Determinísticos (AFD) a partir de um arquivo de entrada contendo a gramática ou a descrição de um Autômato Finito Não Determinístico (AFN). Ele gera arquivos texto com as representações do AFD original, seu complemento e o AFD invertido, além de permitir verificar se uma palavra é aceita pelo AFD.

Classes e Funcionalidades
    1. GeradorDFA
        Responsável por gerar os estados do AFD a partir de um AFN.

        Constrói os estados do AFD, suas transições (rotas) e trata estados "mortos".

        Mantém a estrutura dos estados, símbolos (letras) e regras de transição.

        Usa métodos para ordenar estados, remover caracteres repetidos e identificar novos estados para a construção completa do AFD.

    2. ArquivoDFA
        Recebe o AFD gerado pelo GeradorDFA e organiza as informações para criar a representação textual.

        Define quais estados são iniciais e quais são finais.

        Gera um arquivo texto chamado AFD Original.txt contendo:

        O conjunto de estados

        O alfabeto (símbolos)

        As regras de transição (funções)

        O estado inicial

        Os estados finais

    3. ComplementoDFA
        Cria o complemento do AFD original, invertendo os estados finais para não finais e vice-versa.

        Gera um arquivo texto chamado AFD Complementado.txt com a descrição do autômato complementado.

        Usa os dados do ArquivoDFA para manter o mesmo conjunto de estados e alfabeto.

    4. InversoDFA
        Cria o autômato inverso do AFD original, invertendo as transições.

        Troca os estados finais e iniciais.

        Gera um arquivo texto chamado AFD Invertido.txt.

        Ajusta as transições para que a direção seja invertida.

Classe Principal: Main
    O que faz?
        Instancia os processos de complementação e inversão do AFD, gerando os arquivos correspondentes.

        Lê o AFD original para obter seus estados.

        Solicita ao usuário uma palavra para verificar se é aceita pelo AFD original.

        Exibe na tela se a palavra é aceita ou não.

        Indica quais arquivos foram gerados.

    Como usar?
        Prepare um arquivo texto que contenha a gramática ou a descrição do seu autômato, por exemplo: "Gramática 1.txt".

        Execute o script principal que contém a classe Main, passando o caminho do arquivo como argumento (já feito no código com Main("Gramática 1.txt")).

        Ao rodar o programa, será solicitado que você digite uma palavra para teste.

        O programa responderá se a palavra é aceita ou não pelo autômato.

        Os seguintes arquivos serão gerados automaticamente no diretório onde o script foi executado:

        AFD Original.txt

        AFD Complementado.txt

        AFD Invertido.txt
